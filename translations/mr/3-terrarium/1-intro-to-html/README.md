### परिचय

HTML, म्हणजेच हायपरटेक्स्ट मार्कअप भाषा, वेबचे 'स्नायू' आहे. जर CSS तुमच्या HTML ला सजवते आणि JavaScript त्याला जीवन देते, तर HTML तुमच्या वेब अनुप्रयोगाचे शरीर आहे. HTML चा सिंटॅक्स देखील त्या कल्पनेला प्रतिबिंबित करतो, कारण त्यात "head", "body", आणि "footer" टॅग्स समाविष्ट आहेत.

या धड्यात, आपण HTML वापरून आपल्या आभासी टेरारियमच्या इंटरफेसचा 'स्नायू' तयार करणार आहोत. यात एक शीर्षक आणि तीन स्तंभ असतील: उजवा आणि डावा स्तंभ जिथे ड्रॅग करण्यायोग्य झाडे असतील, आणि मध्यभागी एक क्षेत्र असेल जे खऱ्या काचेसारखे टेरारियम असेल. या धड्याच्या शेवटी, तुम्हाला स्तंभांमध्ये झाडे दिसतील, परंतु इंटरफेस थोडासा विचित्र दिसेल; काळजी करू नका, पुढील विभागात तुम्ही CSS शैली इंटरफेसमध्ये जोडून ते अधिक चांगले दिसेल.

### कार्य

तुमच्या संगणकावर 'terrarium' नावाचा एक फोल्डर तयार करा आणि त्यात 'index.html' नावाची फाइल तयार करा. तुम्ही तुमचा टेरारियम फोल्डर तयार केल्यानंतर नवीन VS कोड विंडो उघडून, 'open folder' वर क्लिक करून आणि तुमच्या नवीन फोल्डरकडे नेव्हिगेट करून Visual Studio Code मध्ये हे करू शकता. एक्सप्लोरर पॅनमधील लहान 'file' बटणावर क्लिक करा आणि नवीन फाइल तयार करा:

![VS कोडमधील एक्सप्लोरर](../../../../translated_images/vs-code-index.e2986cf919471eb984a0afef231380c8b132b000635105f2397bd2754d1b689c.mr.png)

किंवा

तुमच्या git bash वर हे आदेश वापरा:
* `mkdir terrarium`
* `cd terrarium`
* `touch index.html`
* `code index.html` किंवा `nano index.html`

> index.html फाइल्स ब्राउझरला सूचित करतात की ती फोल्डरमधील डिफॉल्ट फाइल आहे; जसे की `https://anysite.com/test` URL फोल्डर संरचनेचा वापर करून तयार केला जाऊ शकतो ज्यामध्ये `test` नावाचा फोल्डर आणि त्यात `index.html` असतो; URL मध्ये `index.html` दिसण्याची गरज नाही.

## डॉकटाइप आणि html टॅग्स

HTML फाइलची पहिली ओळ तिचा डॉकटाइप असते. फाइलच्या अगदी वर ही ओळ असणे आवश्यक आहे हे थोडे आश्चर्यकारक आहे, परंतु हे जुन्या ब्राउझरला सांगते की पृष्ठ सध्याच्या HTML तपशीलांचे अनुसरण करून मानक मोडमध्ये रेंडर करणे आवश्यक आहे.

> टिप: VS कोडमध्ये, तुम्ही टॅगवर होवर करू शकता आणि त्याच्या वापराबद्दल MDN संदर्भ मार्गदर्शकांकडून माहिती मिळवू शकता.

दुसरी ओळ `<html>` टॅगची ओपनिंग टॅग असावी, सध्या त्याच्या क्लोजिंग टॅग `</html>` ने लगेचच पाठपुरावा केला पाहिजे. हे टॅग्स तुमच्या इंटरफेसचे मूळ घटक आहेत.

### कार्य

तुमच्या `index.html` फाइलच्या शीर्षस्थानी या ओळी जोडा:

```HTML
<!DOCTYPE html>
<html></html>
```

✅ डॉकटाइप सेट करून क्वेरी स्ट्रिंगसह काही वेगवेगळे मोड ठरवले जाऊ शकतात: [Quirks Mode आणि Standards Mode](https://developer.mozilla.org/docs/Web/HTML/Quirks_Mode_and_Standards_Mode). हे मोड्स खूप जुने ब्राउझर (Netscape Navigator 4 आणि Internet Explorer 5) समर्थित करण्यासाठी वापरले जात होते जे सध्या सामान्यतः वापरले जात नाहीत. तुम्ही मानक डॉकटाइप घोषणेशी चिकटून राहू शकता.

## दस्तऐवजाचा 'head'

HTML दस्तऐवजाचा 'head' भाग तुमच्या वेब पृष्ठाबद्दल महत्त्वाची माहिती समाविष्ट करतो, ज्याला [metadata](https://developer.mozilla.org/docs/Web/HTML/Element/meta) म्हणून ओळखले जाते. आमच्या बाबतीत, आम्ही वेब सर्व्हरला सांगतो ज्यावर हे पृष्ठ रेंडर केले जाईल, या चार गोष्टी:

-   पृष्ठाचे शीर्षक
-   पृष्ठ मेटाडेटा ज्यामध्ये समाविष्ट आहे:
    -   'character set', पृष्ठात वापरलेले वर्ण एन्कोडिंग सांगते
    -   ब्राउझर माहिती, ज्यामध्ये `x-ua-compatible` समाविष्ट आहे जे सूचित करते की IE=edge ब्राउझर समर्थित आहे
    -   पृष्ठ लोड झाल्यावर व्ह्यूपोर्ट कसे वागावे याबद्दलची माहिती. व्ह्यूपोर्टला प्रारंभिक स्केल 1 असणे सेट केल्याने पृष्ठ प्रथम लोड झाल्यावर झूम स्तर नियंत्रित होतो.

### कार्य

`<html>` टॅगच्या ओपनिंग आणि क्लोजिंग टॅग्सच्या दरम्यान तुमच्या दस्तऐवजामध्ये 'head' ब्लॉक जोडा.

```html
<head>
	<title>Welcome to my Virtual Terrarium</title>
	<meta charset="utf-8" />
	<meta http-equiv="X-UA-Compatible" content="IE=edge" />
	<meta name="viewport" content="width=device-width, initial-scale=1" />
</head>
```

✅ तुम्ही व्ह्यूपोर्ट मेटा टॅग असे सेट केल्यास काय होईल: `<meta name="viewport" content="width=600">`? [व्ह्यूपोर्ट](https://developer.mozilla.org/docs/Web/HTML/Viewport_meta_tag) बद्दल अधिक वाचा.

## दस्तऐवजाचा `body`

### HTML टॅग्स

HTML मध्ये, तुम्ही तुमच्या .html फाइलमध्ये टॅग्स जोडता जेणेकरून वेब पृष्ठाचे घटक तयार करता येतील. प्रत्येक टॅगमध्ये सामान्यतः ओपनिंग आणि क्लोजिंग टॅग असतो, जसे की: `<p>hello</p>` जे परिच्छेद सूचित करते. `<html>` टॅगच्या जोडीच्या आत `<body>` टॅग्सचा संच जोडून तुमच्या इंटरफेसचे शरीर तयार करा; तुमचे मार्कअप आता असे दिसेल:

### कार्य

```html
<!DOCTYPE html>
<html>
	<head>
		<title>Welcome to my Virtual Terrarium</title>
		<meta charset="utf-8" />
		<meta http-equiv="X-UA-Compatible" content="IE=edge" />
		<meta name="viewport" content="width=device-width, initial-scale=1" />
	</head>
	<body></body>
</html>
```

आता, तुम्ही तुमचे पृष्ठ तयार करण्यास सुरुवात करू शकता. सामान्यतः, तुम्ही पृष्ठातील वेगवेगळ्या घटकांसाठी `<div>` टॅग्स वापरता. आम्ही `<div>` घटकांची मालिका तयार करू जी प्रतिमा समाविष्ट करतील.

### प्रतिमा

एक HTML टॅग ज्याला क्लोजिंग टॅगची आवश्यकता नाही तो म्हणजे `<img>` टॅग, कारण त्यात `src` घटक असतो जो आयटम रेंडर करण्यासाठी पृष्ठाला आवश्यक असलेली सर्व माहिती समाविष्ट करतो.

तुमच्या अॅपमध्ये `images` नावाचा फोल्डर तयार करा आणि त्यात [source code folder](../../../../3-terrarium/solution/images) मधील सर्व प्रतिमा जोडा; (झाडांच्या 14 प्रतिमा आहेत).

## सेमॅंटिक मार्कअप

सामान्यतः, HTML लिहिताना अर्थपूर्ण 'सेमॅंटिक्स' वापरणे अधिक चांगले असते. याचा अर्थ काय? याचा अर्थ असा की तुम्ही HTML टॅग्सचा वापर डेटा किंवा संवादाच्या प्रकारासाठी करावा ज्यासाठी ते डिझाइन केले गेले आहेत. उदाहरणार्थ, पृष्ठावरील मुख्य शीर्षक मजकूरासाठी `<h1>` टॅग वापरावा.

तुमच्या ओपनिंग `<body>` टॅगच्या खाली खालील ओळ जोडा:

```html
<h1>My Terrarium</h1>
```

सेमॅंटिक मार्कअप वापरणे जसे की हेडर्ससाठी `<h1>` आणि अनऑर्डर्ड यादी `<ul>` म्हणून रेंडर करणे स्क्रीन रीडर्सना पृष्ठावर नेव्हिगेट करण्यात मदत करते. सामान्यतः, बटणांना `<button>` म्हणून लिहिले पाहिजे आणि यादी `<li>` असावी. जरी क्लिक हँडलर्ससह विशेष शैलीतील `<span>` घटक वापरून बटणे बनवणे _शक्य_ असले तरी, अपंग वापरकर्त्यांसाठी तंत्रज्ञानाचा वापर करून पृष्ठावर बटण कोठे आहे हे ठरवणे आणि त्यावर संवाद साधणे चांगले आहे, जर घटक बटण म्हणून दिसत असेल. याच कारणास्तव, शक्य तितक्या सेमॅंटिक मार्कअप वापरण्याचा प्रयत्न करा.

## टेरारियम

या इंटरफेसचा शेवटचा भाग टेरारियम तयार करण्यासाठी शैलीबद्ध केले जाणारे मार्कअप तयार करणे समाविष्ट करतो.

### कार्य:

शेवटच्या `</div>` टॅगच्या वर हा मार्कअप जोडा:

```html
<div id="terrarium">
	<div class="jar-top"></div>
	<div class="jar-walls">
		<div class="jar-glossy-long"></div>
		<div class="jar-glossy-short"></div>
	</div>
	<div class="dirt"></div>
	<div class="jar-bottom"></div>
</div>
```

✅ जरी तुम्ही स्क्रीनवर हा मार्कअप जोडला असला तरी, तुम्हाला काहीही रेंडर होताना दिसत नाही. का?

## GitHub Copilot Agent Challenge 🚀

Agent मोड वापरून खालील आव्हान पूर्ण करा:

**वर्णन:** टेरारियम प्रकल्पात जोडता येईल अशा प्लांट केअर गाइड विभागासाठी सेमॅंटिक HTML संरचना तयार करा.

**प्रॉम्प्ट:** "प्लांट केअर गाइड" मुख्य शीर्षक, "पाणी देणे", "प्रकाशाची आवश्यकता", आणि "मातीची काळजी" या तीन उपविभागांसह सेमॅंटिक HTML विभाग तयार करा, प्रत्येकामध्ये प्लांट केअर माहितीचा परिच्छेद समाविष्ट आहे. `<section>`, `<h2>`, `<h3>`, आणि `<p>` सारख्या योग्य सेमॅंटिक HTML टॅग्सचा वापर करून सामग्री योग्य प्रकारे संरचित करा.

## 🚀आव्हान

HTML मध्ये काही जुने 'मजेदार' टॅग्स आहेत जे अजूनही वापरण्यास मजेदार आहेत, जरी तुम्ही [या टॅग्स](https://developer.mozilla.org/docs/Web/HTML/Element#Obsolete_and_deprecated_elements) सारख्या अप्रचलित टॅग्सचा तुमच्या मार्कअपमध्ये वापर करू नये. तरीही, तुम्ही जुना `<marquee>` टॅग वापरून h1 शीर्षक आडवे स्क्रोल करू शकता का? (जर तुम्ही केले, तर नंतर ते काढून टाकायला विसरू नका)

## पुनरावलोकन आणि स्व-अभ्यास

HTML ही 'परीक्षित आणि सत्य' बिल्डिंग ब्लॉक प्रणाली आहे ज्याने वेबला आज जे आहे ते बनवण्यास मदत केली आहे. काही जुने आणि नवीन टॅग्सचा अभ्यास करून त्याच्या इतिहासाबद्दल थोडेसे जाणून घ्या. तुम्ही शोधू शकता का की काही टॅग्स का अप्रचलित झाले आणि काही जोडले गेले? भविष्यात कोणते टॅग्स सादर केले जाऊ शकतात?

वेब आणि मोबाइल डिव्हाइससाठी साइट्स तयार करण्याबद्दल अधिक जाणून घ्या [Microsoft Learn](https://docs.microsoft.com/learn/modules/build-simple-website/?WT.mc_id=academic-77807-sagibbon) वर.
